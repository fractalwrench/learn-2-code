# Cheatsheets

## Using the command line

Open or quit a terminal session on Mac by following [these instructions](https://support.apple.com/en-gb/guide/terminal/apd5265185d-f365-44cb-8b09-71a064a42125/mac).

The following commands are essential for navigating around the file system:

| Command  | Description |
| ------------- | ------------- |
| `pwd`  | Prints the current working directory (useful for finding out where you are) |
| `ls`  |  Lists all files in the current working directory  |
| `ls my-directory`  |  Lists all files in the specified directory  |
| `cd my-path`  | Changes the working directory to the given path |
| `vim`  | A command line text editor. This can be exited by following [these instructions](https://stackoverflow.com/a/11828573) |

## Version Control with Git

For a comprehensive cheatsheet see [Atlassian's cheatsheet](https://www.atlassian.com/dam/jcr:e7e22f25-bba2-4ef1-a197-53f46b6df4a5/SWTM-2088_Atlassian-Git-Cheatsheet.pdf
) (PDF).

| Command  | Description |
| ------------- | ------------- |
| `git init`  | Initialises a new git repository |
| `git add <file-name>`  | Adds a file to the Git staging area |
| `git add .`  | Adds all files in the current directory to the Git staging area |
| `git commit -m "Explanation of changes"`  | Commits all changes in the staging area into a commit with the given message   |
| `git init`  | Initialises a new git repository |
| `git push`  | Pushes changes made by you from the local git repository to a remote git repository |
| `git pull`  | Pulls changes made by someone else from a remote git repository to the local git repository |
| `git checkout -b "branch-name"`  | Checks out a new branch from the current branch with the given name |
